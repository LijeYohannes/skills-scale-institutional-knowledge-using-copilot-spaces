# Role Checklists & Handoff Points

This file contains actionable checklists per role to improve clarity, accountability, and handoffs.

## Project Manager (PM) Checklist
- [ ] Create project one-pager and share with stakeholders
- [ ] Maintain project timeline and milestone tracker
- [ ] Keep risk register updated weekly
- [ ] Confirm operational readiness with SRE before release
- [ ] Schedule stakeholder demos and produce status notes

## Product Manager (PdM) Checklist
- [ ] Define and doc success metrics for each feature
- [ ] Prioritize backlog and maintain acceptance criteria
- [ ] Validate designs with UX and user feedback
- [ ] Review experiment designs with Data Analyst
- [ ] Approve release scope with PM and SRE

## UX Designer Checklist
- [ ] Deliver wireframes and prototype for prioritized features
- [ ] Run at least one round of usability testing for major UX changes
- [ ] Provide final assets and specs to Developers
- [ ] Review implemented UI in staging and flag regressions

## Data Analyst Checklist
- [ ] Define metrics and events needed for features
- [ ] Verify instrumentation in staging
- [ ] Produce baseline analysis and post-release review
- [ ] Share dashboards with stakeholders

## SRE Checklist
- [ ] Confirm deployment pipeline and rollback plan
- [ ] Define monitoring and alerts for new releases
- [ ] Validate capacity and scaling plans for major releases
- [ ] Participate in release readiness call

## Scrum Master Checklist
- [ ] Ensure backlog refinement happens regularly
- [ ] Facilitate sprint planning and retrospectives
- [ ] Track team blockers and escalation paths
- [ ] Follow up on retrospective action items

## Handoff Examples
- Design → Dev: deliver prototypes + acceptance criteria + assets
- Dev → QA: provide staging link + test cases + known issues
- PM → Stakeholders: send demo notes + success metrics to track post-release
- PM → SRE: provide deployment date, rollback plan, and monitoring expectations
